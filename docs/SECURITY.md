# 安全策略

这个仓库是一个 **Claude Code 本地重建与修复工作区**，不是 Anthropic 官方生产服务，也不是官方漏洞提交通道。

因此，本文件的作用是明确：

- 这个仓库的安全问题应如何处理；
- 哪些问题属于本仓库范围；
- 哪些问题应提交给其他上游或服务提供方。

---

## 1. 适用范围

本仓库安全处理的重点是：

- 本仓库源码中的本地构建与运行逻辑；
- 本仓库新增的 shim、stub、兼容层和构建脚本；
- 本仓库文档中可能导致明显误配置、误暴露或危险使用方式的内容；
- 这个重建工作区本身引入的本地文件、路径、命令执行或配置处理风险。

---

## 2. 不属于本仓库直接处理的内容

以下问题通常不属于这个重建仓库本身的直接安全范围：

1. Anthropic 官方在线服务或官方生产基础设施
2. 第三方代理、兼容网关或自建 `ANTHROPIC_BASE_URL` 服务本身的漏洞
3. 外部依赖的上游漏洞
4. 用户本地机器已经存在的系统级安全问题
5. 用户自行暴露的 API key、代理凭据或私有配置

如果问题发生在这些范围内，通常应该报告给对应的实际维护方，而不是只报告到这个仓库。

---

## 3. 报告建议

### 3.1 非敏感问题

如果问题：

- 不涉及密钥泄露；
- 不涉及远程代码执行；
- 不涉及高危权限绕过；
- 不包含可直接复用的攻击细节；

那么可以直接通过 GitHub issue 报告。

### 3.2 敏感问题

如果问题涉及：

- 凭据泄露；
- 任意命令执行；
- 权限绕过；
- 路径穿越；
- 危险默认配置导致的高风险利用；

请不要先公开完整 exploit 细节。

更安全的做法是：

1. 优先使用 GitHub Security Advisories（如果仓库已启用）；
2. 如果尚未启用，请先联系仓库维护者建立私下沟通渠道；
3. 在无法立即建立私下通道时，公开 issue 里只写最小必要信息，不贴 exploit、密钥、payload 或可直接复现的攻击脚本。

---

## 4. 报告时请尽量提供什么

一份有用的安全报告通常应包含：

- 受影响的文件或模块路径
- 触发条件
- 影响范围
- 最小复现步骤
- 是否需要特定环境变量、代理或配置
- 你的风险判断
- 你建议的修复方向（如果有）

如果问题涉及本地文件、环境变量或密钥，请先做脱敏。

---

## 5. 对外披露建议

在敏感问题修复前，建议不要公开：

- 可直接复制利用的 payload
- 真实 API key
- 用户本地路径中的敏感信息
- 可帮助第三方立刻复现攻击的完整 PoC

修复完成后，再补充公开说明、版本边界和缓解建议会更合适。

---

## 6. 对维护者的建议

如果后续准备把这个仓库正式公开运营，建议尽快补齐：

1. GitHub Security Advisories
2. 明确的私下安全联系渠道
3. 一份更正式的响应 SLA
4. dependency update / advisory review 流程

在这些基础设施没准备好之前，请不要在 README 里把这个仓库描述为生产级官方分发版本。

---

## 7. 当前最重要的安全边界提醒

这个仓库当前已经可以启动 full local CLI 并进入 REPL，但这不代表：

- 认证、代理和网络环境默认就是安全的；
- 任意第三方 `ANTHROPIC_BASE_URL` 都值得信任；
- 本地配置、日志和调试输出中不会包含敏感信息。

如果你在调试模式下运行：

```bash
bun ./dist/full/cli.js --debug-to-stderr
```

请特别注意不要把包含密钥、代理地址、请求头或本地敏感路径的日志直接公开贴出。
